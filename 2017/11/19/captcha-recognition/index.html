驗證碼

常見的驗證碼
又稱captcha，目的是辨別使用者和機器人，
推文中主要是討論台鐵 和台灣證券交易所的固定長度驗證碼。

如何破解驗證碼

想要自動產生答案，是一種圖片轉文字的問題，
也就是影像識別，這是個研究許多年的領域，過去有許多方法處理相關的 task，
不過近年來深度學習（Deep Learning）興起，
這篇只會討論如何利用深度學習來解決這個 task。

Multi-Class Convolution Neural Network

Architecture of Convolutional Neural Network
Convolution Neural Network 簡稱 CNN，近年來非常流行的方法，
我就不多作介紹了。

使用CNN來對圖片分類，此時有一個問題很重要，
「切字」

切字
把每個字切開，可以減少 class 的數量，
若 captcha 只有數字，class 數量是 10 個，
若 captcha 包含英文，class 數量是 10+26(+26) 個，
不過有些圖片並不是這麼好切字，
這點則是切字法的最大缺點。

不切字
不切字把整張圖當作 model 的 input，
如果 captcha 有五個字，
class 數量就有 105 或 625，
105或許還能 train，625就有些太誇張了，
推文中，bluecadence 提到他能夠不切字並且辨識率突破 99%，
這點讓我很驚訝，也是我對這篇文感興趣的原因。

CNN Encoder and RNN decoder

看到這個 task，我一開始的想法便是這個方法，
先用 CNN 將圖片 encode 成一串 vector，
再由 RNN 一步一步把每個字解析出來。

但我認為這個方法需要更多的 training data，
如果手頭上沒有足夠多的 data，
一般人不太適合使用這種 model。

另外，此方法應用非常廣泛，
這也是現今常見的 image-to-text 常見的 model，
如下圖，常見應用是產生圖片標題。

Multi-Label Convolution Neural Network

終於來到本篇的重頭戲，
既然bluecadence大大只用了簡單的 CNN ，就能有良好效果，
那麼肯定是 model 設計上有一些巧思，

我在 Github 上查到許多關於 verification code / captcha 的 repo，
CharlesLiuyx的這個專案最符合bluecadence大的說法，
一樣只用了簡單的 CNN 和 MLP 分類，
下面引用了關鍵部分程式碼。

簡單來說就是將答案轉為長度 63 * 4 的 vector (由 0 和 1 組成)，
每63個數字代表一個「字」，
底線則代表空答案，可以處理不同長度的 training data，
對 model 來說，每個 element 都可以是 0 或 1，變成 multi-label 的問題，
這麼一來，model 的 output 種類也不會像先前如此誇張，
是個值得思考的好設計XD

另外需要注意的是，
multi-label 的 loss function 和平常的 multi-class 不同，
以 tensorflow 為例，loss function 要改為 tf.nn.sigmoid_cross_entropy_with_logits，
原因就自己去查吧 :p

增加 Training Data

文章中，f496328mm提到他的方法，

可以利用台鐵驗證碼系統，幫我上label啦，
簡單講，對或錯台鐵會幫我判定，對的就存label，這樣就變成自動上label啦，data會越來越多，有點類似reinforcement learning

這個想法真的非常棒，
不僅同時測試model也同時增加資料量。

驗證碼還有用嗎?
